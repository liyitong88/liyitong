# 医学装备科任务闭环管理系统部署说明

这是一个前端静态应用，当前最实用的发布方式有三种。

## 1. 本机直接打开

适合电脑本地测试。

```bash
pnpm install
pnpm dev
```

打开终端里显示的地址即可。

## 2. 手机在同一局域网访问

适合电脑和手机连同一个 Wi-Fi。

```bash
pnpm dev:host
```

然后在手机浏览器输入电脑显示的局域网地址，例如：

```text
http://电脑IP:5173/
```

如果 5173 被占用，也可以用生产预览：

```bash
pnpm build
pnpm preview:host
```

然后在手机打开：

```text
http://电脑IP:4174/
```

## 3. 手机在外网访问

如果不在同一个 Wi-Fi，最省事的方式是用临时外网隧道：

```powershell
start-mobile.bat
```

或者直接双击项目根目录下的 `start-mobile.bat`。

启动后会显示一个 `https://...trycloudflare.com` 地址，手机直接打开它即可。

注意：

- 这是临时地址，关闭启动窗口后会失效
- 如果想要长期固定地址，需要把项目部署到正式静态托管平台

## 4. 正式部署到 Vercel 或 Netlify

仓库已经准备好静态托管配置：

- `vercel.json`
- `netlify.toml`

部署时只需要：

1. 推送代码到 Git 仓库
2. 在 Vercel 或 Netlify 导入仓库
3. 构建命令填写 `pnpm build`
4. 输出目录填写 `dist`

这类平台会给你一个长期稳定的公开地址，适合真正交付使用。

## 5. 交付前建议

正式给别人使用前，建议再检查：

1. 默认管理员账号是否修改
2. 手机端新增任务是否顺手
3. 任务保存和返回是否正常
4. 数据备份和导入是否可用
