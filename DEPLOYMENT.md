# 医学装备科任务闭环管理系统部署说明

这是一个前端静态应用，部署思路和“月度绩效核算工具”一致：

- 先用本地浏览器存储跑通
- 再通过导出备份 / 导入恢复兜底
- 最后用静态托管拿到长期公开地址

当前最实用的发布方式有三种。

## 1. 本机直接打开

适合电脑本地测试。

```bash
pnpm install
pnpm dev
```

打开终端里显示的地址即可。

## 2. 手机在同一局域网访问

适合电脑和手机连同一个 Wi-Fi。

```bash
pnpm dev:host
```

然后在手机浏览器输入电脑显示的局域网地址，例如：

```text
http://电脑IP:5173/
```

如果 5173 被占用，也可以用生产预览：

```bash
pnpm build
pnpm preview:host
```

然后在手机打开：

```text
http://电脑IP:4174/
```

## 3. 手机在外网访问

如果你不在同一个 Wi-Fi，临时方案仍然可以用：

```powershell
start-mobile.bat
```

或者直接双击项目根目录下的 `start-mobile.bat`。

启动后会显示一个 `https://...trycloudflare.com` 地址，手机直接打开它即可。

注意：

- 这是临时地址，关闭启动窗口后会失效
- 它适合调试，不适合正式交付

## 4. 正式部署到 Cloudflare Pages

如果你的目标是“换网络、换地点、手机电脑都尽量能打开”，推荐把它部署到 **Cloudflare Pages**，并绑定自己的**自定义域名**。

Cloudflare Pages 的官方文档说明了两种适合这种项目的方式：

- 上传预构建产物进行部署
- 绑定自定义域名

推荐流程：

1. 先执行 `pnpm build`
2. 把 `dist` 作为发布产物上传到 Cloudflare Pages
3. 给项目绑定一个自己的域名
4. 以后都用这个固定地址访问

这样得到的是长期公开地址，不依赖你电脑开着，也不依赖同一个 Wi-Fi。

## 5. 这类项目的约定

如果后面继续沿用“月度绩效核算工具”的方式做迭代，建议保持这三个约定：

1. 页面尽量保持单页静态应用
2. 业务数据尽量留在浏览器本地
3. 每次调整都同步补一份备份导入导出

## 6. 交付前建议

正式给别人使用前，建议再检查：

1. 默认管理员账号是否修改
2. 手机端新增任务是否顺手
3. 任务保存和返回是否正常
4. 数据备份和导入是否可用
